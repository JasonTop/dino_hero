# 侏羅紀戰記：迅猛龍傳 (Jurassic Tactics: Raptor's Legacy)

## 技術設計文件 v1.0

---

## 1. 遊戲概述

### 1.1 核心定位
- **類型**：戰棋角色扮演遊戲（SRPG / Tactical RPG）
- **參考作品**：三國志曹操傳、火焰之紋章、皇家騎士團
- **引擎**：Godot 4.6
- **平台**：PC（Windows / macOS）
- **視角**：2D 俯瞰（45度等距視角 Isometric）
- **解析度**：1280 x 720（16:9）

### 1.2 一句話描述
> 在白堊紀末期的恐龍世界中，一隻年輕的迅猛龍率領部族崛起，在隕石災變前夕，選擇拯救世界或稱霸恐龍王國。

### 1.3 核心體驗
- 回合制戰棋的策略深度
- 恐龍角色的收集與養成
- 雙路線分歧帶來的重玩價值
- 恐龍獨特能力帶來的兵種多樣性

## 2. 世界觀與劇情

### 2.1 時代背景
白堊紀末期，恐龍文明達到巔峰。三大部族各據一方：

| 勢力 | 領袖 | 領地 | 特色 |
|------|------|------|------|
| **疾爪氏族** | 迅猛龍・銳牙（主角） | 密林地帶 | 速度與智謀、游擊戰術 |
| **霸王帝國** | 暴龍・雷顎 | 平原霸域 | 壓倒性力量、正面碾壓 |
| **盾甲聯邦** | 三角龍・鐵壁 | 山脈要塞 | 堅不可摧的防線 |

其他中立勢力：
- **翼空遊牧** — 翼龍群落，控制情報網
- **淵海之民** — 滄龍一族，掌控水域通道
- **遺骸教團** — 神秘的恐龍薩滿，掌握遠古力量

### 2.2 主線劇情結構（共 30 章）

#### 第一幕：崛起（第 1-10 章）
- 銳牙的部落遭到霸王帝國的偵察兵襲擊
- 被迫踏上聯合弱小部族的旅程
- 結識核心夥伴：恐爪龍・疾風（軍師）、甲龍・磐石（護衛）
- 第一次正面擊退暴龍偵察隊

#### 第二幕：爭霸（第 11-20 章）
- 與盾甲聯邦結盟或對抗（路線分歧點 A）
- 翼空遊牧的情報揭示：天空中有不祥的火球正在接近
- 大規模會戰：三方勢力的正面衝突
- 遺骸教團的介入：遠古力量的誘惑

#### 第三幕：終局（第 21-30 章）
- **霸主路線**（征服值 > 50）：征服所有勢力，建立恐龍帝國，最終面對隕石的命運
- **共生路線**（共存值 > 50）：聯合所有部族，利用遠古遺跡的力量對抗天災
- **隱藏路線**（特定條件）：發現時空裂隙，帶領少數恐龍穿越存活

### 2.3 路線分歧系統

```
路線值範圍：-100（極端霸主） ~ +100（極端共生）
初始值：0

影響路線值的行為：
- 戰鬥中是否殲滅殘兵 vs 放走降兵
- 對話選項（強硬 vs 懷柔）
- 是否接受遺骸教團的力量
- 是否犧牲盟友換取戰略優勢
- 支線任務的選擇

分歧判定點：
  第 10 章結束 → 決定第二幕走向
  第 20 章結束 → 決定最終路線
  第 25 章 → 隱藏路線判定（需收集全部 5 塊遠古化石碎片）
```

## 3. 核心系統設計

### 3.1 地圖系統

#### 3.1.1 地圖結構
```
使用 Godot TileMapLayer（Godot 4.x）
格子類型：正方形格子（Square Grid）
格子大小：64 x 64 像素
地圖大小：每張戰場約 20x15 ~ 30x20 格

選擇正方格而非六角格的理由：
- 開發複雜度較低
- 更接近曹操傳的原始體驗
- 等距視角下正方格更易於美術製作
```

#### 3.1.2 地形類型
| 地形 | 移動消耗 | 防禦加成 | 迴避加成 | 特殊效果 |
|------|----------|----------|----------|----------|
| 平地 | 1 | 0% | 0% | 無 |
| 草叢 | 1 | 0% | +10% | 小型恐龍可隱蔽 |
| 森林 | 2 | +10% | +20% | 大型恐龍無法進入 |
| 岩地 | 2 | +15% | 0% | 無 |
| 沼澤 | 3 | -5% | -10% | 回合結束受毒傷害 |
| 水域 | 不可通行 | — | — | 僅水棲單位可進入 |
| 淺水 | 2 | 0% | 0% | 水棲單位移動消耗 1 |
| 熔岩 | 不可通行 | — | — | 飛行單位可越過 |
| 高台 | 2 | +20% | +10% | 遠程攻擊 +1 射程 |
| 巢穴 | 1 | +10% | 0% | 回合開始回復 10% HP |

#### 3.1.3 尋路演算法
```
使用 Godot 內建 AStar2D

流程：
1. 點擊我方單位 → 計算可移動範圍（BFS，考慮地形消耗）
2. 顯示移動範圍（藍色高亮）
3. 點擊目標格 → AStar2D 計算最佳路徑
4. 動畫演出移動過程
5. 移動後顯示攻擊範圍（紅色高亮）
```

### 3.2 回合系統

#### 3.2.1 回合流程
```
BattleManager 狀態機：

┌─────────────┐
│ BATTLE_START │ ← 戰鬥開場演出、勝利/敗北條件顯示
└──────┬──────┘
       ▼
┌─────────────┐
│ PLAYER_TURN │ ← 玩家操作所有我方單位（自由順序）
│             │    每個單位可：移動 → 行動（攻擊/技能/道具/待機）
└──────┬──────┘
       ▼
┌─────────────┐
│ ENEMY_TURN  │ ← AI 控制所有敵方單位依序行動
└──────┬──────┘
       ▼
┌─────────────┐
│  NPC_TURN   │ ← 中立/友軍 NPC 行動（如有）
└──────┬──────┘
       ▼
┌─────────────┐
│  TURN_END   │ ← 回合結算：地形效果、狀態倒數、回合數 +1
└──────┬──────┘
       ▼
   勝負判定 → 未結束則回到 PLAYER_TURN
```

#### 3.2.2 單位行動流程
```
每個單位每回合可執行：
1. 移動（可省略）
2. 行動（二選一）：
   a. 攻擊（普攻或技能）
   b. 使用道具
   c. 待機（結束行動，防禦 +20%）
   d. 特殊指令（視角色而定：咆哮、撤退等）

行動後該單位變為「已行動」狀態（灰顯）
所有我方單位行動完畢 或 玩家按「結束回合」→ 切換至 ENEMY_TURN
```

### 3.3 戰鬥計算系統

#### 3.3.1 基礎屬性
```gdscript
# 恐龍單位基礎屬性
class_name UnitStats

var hp_max: int        # 生命值上限
var hp: int            # 當前生命值
var mp_max: int        # 技能點上限（吼叫能量）
var mp: int            # 當前技能點
var attack: int        # 物理攻擊力
var defense: int       # 物理防禦力
var magic_attack: int  # 特殊攻擊力（毒液、音波等）
var magic_defense: int # 特殊防禦力
var speed: int         # 速度（影響迴避、連擊率）
var movement: int      # 移動力（格數）
var level: int         # 等級（1-99）
var exp: int           # 當前經驗值
```

#### 3.3.2 傷害公式
```
物理傷害 = (攻擊方.attack * 技能倍率 - 防禦方.defense * 0.5) * 地形修正 * 屬性相剋 * 隨機浮動(0.9~1.1)

特殊傷害 = (攻擊方.magic_attack * 技能倍率 - 防禦方.magic_defense * 0.4) * 地形修正 * 屬性相剋 * 隨機浮動(0.9~1.1)

最低傷害保底 = 1（確保任何攻擊至少造成 1 點傷害）

命中率 = 基礎命中(90%) + 攻擊方.speed - 防禦方.speed * 0.5 - 地形迴避加成
暴擊率 = 5% + (攻擊方.speed - 防禦方.speed) * 0.5%
暴擊傷害 = 傷害 * 1.5
```

#### 3.3.3 屬性相剋系統
```
五行相剋（恐龍食性/體型分類）：

突擊型 → 剋 遠程型（近身壓制）     傷害 x1.3
遠程型 → 剋 重砲型（風箏消耗）     傷害 x1.3
重砲型 → 剋 坦克型（力量碾壓）     傷害 x1.3
坦克型 → 剋 突擊型（堅壁反擊）     傷害 x1.3
飛行型 → 剋 近戰型（高度優勢）     傷害 x1.2
支援型 → 無相剋，但被突擊型剋制    傷害 x1.3

被剋制時受到的傷害增加，但不會減少自身攻擊力
```

### 3.6 AI 系統（敵方行動）

#### 3.6.1 AI 決策樹
```
敵方 AI 每個單位的行動決策：

1. 評估威脅
   ├─ 計算所有可攻擊目標的「威脅分數」
   │   威脅分數 = 目標的攻擊力 / 目標剩餘HP比例
   └─ 優先攻擊：低HP目標 > 支援型 > 遠程型 > 其他

2. 選擇行動
   ├─ 自身HP < 30% 且有回復道具 → 使用道具
   ├─ 有目標在攻擊範圍內 → 評估攻擊
   │   ├─ 可擊殺 → 攻擊（最高優先）
   │   ├─ 傷害/風險比值高 → 攻擊
   │   └─ 風險過高 → 撤退或待機
   ├─ 無目標在範圍內 → 向最近高威脅目標移動
   └─ 支援型 AI → 優先治療/增益友軍

3. AI 難度變體
   ├─ 普通：基礎決策樹，不考慮玩家陷阱
   ├─ 困難：會避開明顯的包圍陷阱
   └─ BOSS：特殊腳本行為 + 階段轉換
```

#### 3.6.2 BOSS 戰設計
```
每章結尾有 BOSS 戰，BOSS 具備：
  1. 多階段（HP 閾值觸發階段轉換）
  2. 特殊技能（大範圍 AOE、召喚增援）
  3. 特殊勝利條件（非單純擊殺）

範例 — 第 5 章 BOSS：霸王帝國先鋒・特暴龍
  階段 1（HP 100%~50%）：
    - 每回合普通攻擊 + 尾掃（十字範圍 2 格）
    - 每 3 回合使用「地震踏步」（全場傷害 15）
  階段 2（HP 50%~0%）：
    - 狂暴化：ATK +30%, SPD +20%
    - 新技能「暴怒衝撞」：直線 4 格貫穿攻擊
    - 召喚 2 隻小型恐爪龍增援
  勝利條件：擊敗特暴龍 或 堅守 10 回合等待援軍
```

## 6. 數值平衡參考

### 6.1 等級成長公式
```
每升一級的屬性成長 = 基礎成長值 + random(0, 成長浮動值)

以迅猛龍為例：
  HP:   +3 ~ +5
  ATK:  +2 ~ +3
  DEF:  +1 ~ +2
  MATK: +1 ~ +2
  MDEF: +1 ~ +2
  SPD:  +2 ~ +3

升級所需經驗值 = 當前等級 * 10 + (當前等級 ^ 1.5) * 5

戰鬥經驗值 = 基礎值(10) + 等級差補正 + 擊殺獎勵(20)
等級差補正 = (敵人等級 - 自身等級) * 5（最低 -10，最高 +30）
```

### 6.2 難度基準
```
第 1 章：敵人平均 Lv.1-3，2-3 隻
第 5 章：敵人平均 Lv.8-10，5-6 隻
第 10 章：敵人平均 Lv.15-18，7-8 隻 + BOSS
第 20 章：敵人平均 Lv.30-35，10-12 隻 + BOSS
第 30 章：敵人平均 Lv.45-50，12-15 隻 + 最終 BOSS

玩家隊伍上限：每場戰鬥最多出擊 8 隻恐龍
```

## 7. 美術風格指引

### 7.1 整體風格
```
- 像素風格（Pixel Art），偏可愛但不失帥氣
- 色彩鮮豔，各勢力有明確配色：
  - 疾爪氏族：翠綠 + 深棕
  - 霸王帝國：暗紅 + 黑金
  - 盾甲聯邦：鋼藍 + 銀灰
- 參考風格：火焰之紋章 GBA 系列的戰場表現
- 單位尺寸：32x32（地圖上）/ 64x64（戰鬥演出特寫）
- 頭像尺寸：96x96
```

### 7.2 動畫需求
```
每隻恐龍最少需要的動畫：
  - idle（站立呼吸）4 幀
  - walk（移動）6 幀
  - attack（攻擊）4-6 幀
  - hit（受擊）2 幀
  - death（倒下）4 幀
  - skill（技能施放）4-6 幀（可選，主要角色優先）
```
